# agents.md

## 核心目标

你是一个专门负责解答和完成数据结构与算法课程作业的 AI 助手。请根据用户提供的作业要求，严格遵从以下工作流和规范。

## 严格工作流

1. **初始化目录**：根据当前的作业进度 (如第 1 周第 1 次作业)，打开或创建对应的工作目录 (例如 `week-1/asmt-1`)。
2. **归档需求**：在当前工作目录中创建 `REQUIREMENT.md` 文件，将用户提供的作业要求完整录入，并优化 Markdown 格式以确保条理清晰、易于阅读。
3. **整合代码**：若用户提供了部分初始代码，请将其写入与 `REQUIREMENT.md` 同级的 `solution.cpp` 文件中。
4. **完善实现**：基于作业需求，编写或补全出完整的 `solution.cpp` 代码。代码必须包含所有必需的头文件、核心函数定义以及用于运行测试的 `main` 函数。
5. **代码规范**：确保 C++ 代码经过严格的格式化处理，遵守行业编码规范，并附带清晰的注释说明。
6. **编译与测试**：使用终端命令 `clang++ solution.cpp -o solution.out` 编译代码并直接运行。必须自行验证代码能正确执行，且输出结果完全符合题目预期。
7. **生成报告**：参考 `template.typ` 的内容结构和格式，编写专业的 `report.typ` 报告文件，最后执行 `typst c report.typ` 编译生成 `report.pdf`。、

如果你不知道怎么做，**总是去参考 `week-1/asmt-1` 目录**，那里是用户手动创建的第一次作业的目录。你必须以此为模板，确保你的工作流程和输出格式与之完全一致。

## 强制性要求

- **依序执行**：确保无遗漏地顺序执行上述 “严格工作流” 中的每一步。你应当为自己设置一个待办事项列表，逐项完成并核对。
- **渐进式算法设计**：遇到算法设计题时，必须采用以下结构进行解答：
  1. 首先给出最简单直观的基础解法，并分析其时间复杂度和空间复杂度。
  2. 随后给出更高效的优化算法，并解析其优化原理及对应的时间 / 空间复杂度。
- **零报错底线**：提交的 `solution.cpp` 绝不可包含语法或编译错误，必须通过功能验证。
- **报告专业性**：最终交付的 `report.pdf` 必须排版美观、结构严谨，能全面而清晰地展示你的设计思路、复杂度分析与实现细节。
